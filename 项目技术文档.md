## 1. 项目概览

"小爪"是一个暖色治愈风的宠物管家应用，目标平台为 **iOS + Android + 鸿蒙**。当前以 Flutter 为移动端框架，配套 NestJS 后端和 React 管理后台。

主要功能：宠物档案管理、疫苗/驱虫提醒、记账、食谱、记事、体重追踪、病历、囤货库存、AI 宠物管家、成长相册、商城、智能设备管理、SPA 预约等。

## 4. 本地运行

### Flutter 移动端

```bash
cd mobile
flutter pub get
flutter analyze
flutter run -d chrome          # 浏览器
flutter run -d ios             # iOS 模拟器
flutter run -d android         # Android 模拟器
```

### Express 开发服务器 + 管理后台

```bash
npm run dev                    # 启动 Express + Vite，端口 3000
```

管理后台访问：`http://localhost:3000`

### NestJS 后端

```bash
cd backend
npm install
docker compose up -d           # PostgreSQL + Redis + MinIO
npx prisma db push
npm run dev                    # 端口 3000，Swagger: /api/docs
```

## 8. 状态管理

核心文件：`mobile/lib/services/app_provider.dart`

`AppProvider extends ChangeNotifier` 管理：

- 登录状态、宠物列表、当前宠物
- 注册流程中的临时宠物 (`_pendingPet`)
- 提醒 / 账单 / 食谱 / 记事 / 体重记录 / 病历 / 囤货

关键方法：

- `startNewPet()` → `updatePendingPet()` → `completeRegistration()` — 建档流程
- `loginWithFirstPet(Pet)` — 登录并设置宠物
- `switchPet(Pet)` — 切换当前宠物
- `updatePet(Pet)` — 编辑宠物信息
- `seedDemoData()` — 注入演示数据
- `logout()` — 退出并清空状态

---

## 9. AI 系统

Express 服务器 (`server.ts`) 内置 DeepSeek API 集成：

- **意图分类器** — 本地关键词匹配（强宠物词 / 弱宠物词 / 非宠物词 / 紧急词），不消耗 API 额度
- **Prompt 构建** — 拼接宠物档案信息（品种、年龄、体重、病史）生成上下文提示词
- **输出后处理** — Markdown 剥离、停止词截断
- **智能降级** — API 不可用时返回预设回复

## 10. 数据库设计

Prisma Schema（`backend/prisma/schema.prisma`）定义 14 张表：

`User` → `Pet` → `Reminder`, `Expense`, `Recipe`, `Note`, `WeightRecord`, `MedicalRecord`, `StockItem`, `AlbumPhoto`  
`Product` → `CartItem`, `Order` + `OrderItem`  
`RefreshToken`, `Admin`

所有用户数据级联删除。当前开发阶段使用 Express 内存 mock。

## 11. 构建与部署注意事项

- `flutter analyze` 零错误零警告
- `flutter run -d chrome` 可直接在浏览器中调试
- 中文路径可能导致 `flutter build web` 的 shader 输出失败，可复制到无中文短路径构建
- NestJS 需要先启动 Docker 容器（PostgreSQL + Redis + MinIO）
- 管理后台构建：`npm run build`，输出到 `dist/`

## 13. 接手建议

优先阅读：

1. `mobile/lib/main.dart` — 路由和导航
2. `mobile/lib/services/app_provider.dart` — 状态管理
3. `mobile/lib/theme/app_theme.dart` — 设计系统组件
4. `mobile/lib/pages/home_page.dart` — 首页（最复杂页面）
5. `server.ts` — API 和 AI 逻辑

后续产品化建议：

- Flutter 端接入 `server.ts` API（当前使用 AppProvider 本地数据）
- 增加本地持久化（`shared_preferences` 或 SQLite）
- 相册接入真实图片选择和上传
- AI 助手接入 DeepSeek API（当前为演示界面）
- 为鸿蒙平台配置 `flutter_ohos` 引擎
- 为关键状态方法补单元测试
